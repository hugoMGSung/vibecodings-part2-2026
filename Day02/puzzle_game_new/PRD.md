## 프로젝트: 단계형 퍼즐 게임

### 1. 프로젝트 개요
- 브라우저에서 실행되는 퍼즐 맞추기 게임을 만든다.
- 사용자는 시작 화면에서 게임을 시작하고, 퍼즐을 풀면서 여러 단계를 순서대로 진행한다.
- 모든 단계를 완료하면 종료 화면에서 축하 메시지를 확인한다.

### 2. 목표
- 퍼즐 게임의 기본 재미를 유지하면서, 단계 진행과 시간 제한으로 긴장감을 높인다.
- 실제 사용시스템처럼 폴더구성도 하고, HTML, CSS, JavaScript를 독립적인 파일로 만든다.
- 화면 전환, 점수, 시도 횟수, 타임아웃을 포함한 완성형 미니게임을 구현한다.

### 3. 게임 흐름
1. 페이지를 열면 시작 화면이 먼저 보인다.
2. 사용자는 `시작하기` 버튼을 눌러야 게임을 시작할 수 있다.
3. 게임은 `쉬움 -> 중간1 -> 중간2 -> 중간3 -> 어려움` 순서로 자동 진행된다.
4. 각 단계를 제한 시간 안에 모두 맞추면 다음 단계로 이동한다.
5. 마지막 단계까지 완료하면 종료 화면과 축하 메시지를 보여준다.
6. 시간 내에 실패하면 게임 오버가 된다.

### 4. 주요 기능
#### 4-1. 시작 화면
- 첫 진입 시 게임 보드 대신 시작 화면을 표시한다.
- `시작하기` 버튼을 눌러야 보드가 나타난다.
- 시작 화면에는 게임 설명과 도전 안내 문구를 넣는다.

#### 4-2. 퍼즐 보드
- 카드 형태의 퍼즐 보드를 표시한다.
- 카드는 클릭하면 뒤집히고, 같은 그림 2장을 맞추면 고정된다.
- 이미 맞춘 카드는 다시 클릭할 수 없다.
- 매 단계마다 보드 크기를 다르게 사용한다.

#### 4-3. 단계 시스템
- 단계는 총 4개로 구성한다.
- 쉬운 난이도부터 자동으로 시작한다.
- 클리어할 때마다 다음 단계로 자동 이동한다.
- 단계 구성:
  - 쉬움: 2x2
  - 중간1: 4x4
  - 중간2: 6x6
  - 중간3: 8x8
  - 어려움: 10x10

#### 4-4. 타임아웃
- 각 단계마다 제한 시간을 둔다.
- 제한 시간 내에 완료하지 못하면 게임 오버가 된다.
- 남은 시간을 화면에 표시한다.

#### 4-5. 점수
- 같은 그림 한 쌍을 맞출 때마다 10점씩 증가한다.
- 한단계를 넘어가면 보너스점수로 100점을 증가한다/
- 점수는 화면에 실시간 표시한다.
- 게임을 다시 시작하면 점수는 0점으로 초기화한다.

#### 4-6. 시도 횟수
- 두 번째 카드를 선택할 때마다 시도 횟수를 1 증가시킨다.
- 시도 횟수는 화면에 실시간 표시한다.

#### 4-7. 종료 화면
- 모든 단계를 완료하면 축하 종료 화면을 표시한다.
- 최종 점수와 시도 횟수를 안내한다.
- `처음부터 다시하기` 버튼으로 다시 시작할 수 있다.

### 5. 화면 구성
#### 5-1. 공통 영역
- 게임 제목
- 안내 문구
- 시도 횟수
- 점수
- 남은 시간
- 다시 시작 버튼

#### 5-2. 시작 화면
- 시작 안내 문구
- 게임 설명
- `시작하기` 버튼

#### 5-3. 종료 화면
- 축하 문구
- 최종 결과 요약
- `처음부터 다시하기` 버튼

### 6. 규칙
- 카드 2장을 선택했을 때 같은 그림이면 성공 처리한다.
- 다른 그림이면 잠시 보여준 뒤 다시 뒤집는다.
- 게임 진행 중에는 입력 충돌이 나지 않도록 보드를 잠시 잠근다.
- 시작 전에는 카드 클릭이 불가능해야 한다.
- 종료 후에는 카드 클릭이 불가능해야 한다.

### 7. 데이터와 상태
- 게임 상태는 브라우저 메모리에서만 관리한다.
- 저장소(DB, 서버)는 사용하지 않는다.
- 관리해야 할 상태 예시:
  - 현재 단계
  - 시도 횟수
  - 점수
  - 남은 시간
  - 선택한 카드 2장
  - 맞춘 카드 수
  - 게임 시작 여부
  - 게임 종료 여부

### 8. 기술 스택

### 9. 비기능 요구사항

- 화면 전환과 카드 동작이 자연스럽게 느껴져야 한다.
- 코드 변경 시 기능별로 분리된 구조를 유지한다.
